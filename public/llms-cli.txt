# 7onic Design System ‚Äî AI Guide (CLI)

## What is 7onic?

Token-driven React design system with 42 components on Radix UI.
Single source of truth ‚Äî design tokens to code. Use this to build any React service.

- Documentation: https://7onic.design
- CLI: `npx 7onic init` + `npx 7onic add <component>` (local file copy)
- npm: `@7onic-ui/tokens` (design tokens, installed by CLI)
- For tokens-only guide, see: https://7onic.design/llms.txt
- For npm package guide, see: https://7onic.design/llms-full.txt

## How to Start

### Step 1: Ask the user (setup checklist)

Before writing any code, present this checklist and wait for answers:

1. **Framework** ‚Äî Next.js / Vite (React SPA) / Remix / other?
2. **Dark mode** ‚Äî Yes or no?
3. **Font** ‚Äî Use default, or custom font? If custom, which font?
4. **Locale** ‚Äî Which language(s)? (for CJK font loading)
5. **What are you building?** ‚Äî Describe the project

**If the user answers in natural language** (e.g., "Make me a dashboard with dark mode, English only"), extract the answers from their message. **If any item is missing, ask a follow-up question for the missing items only.** Do not proceed until all 5 items are answered.

### Step 2: Initialize project with CLI

Run the 7onic CLI to set up the project:

```bash
npx 7onic init
```

This automatically:
- Detects Tailwind v3/v4
- **Vite projects**: configures `@/` path alias in `tsconfig.app.json` and `vite.config.ts`, installs `@types/node`
- Installs base dependencies (@7onic-ui/tokens, lucide-react, class-variance-authority, clsx, tailwind-merge)
- Sets up globals.css with all required token imports (v4: auto-injects `@import "tailwindcss"` if missing)
- Creates `cn()` utility at `lib/utils.ts`
- Generates `7onic.json` config

**After init, apply user's answers:**

- Dark mode = yes ‚Üí implement dark mode toggle (see below)
- Custom font ‚Üí load font (next/font for Next.js, CDN for Vite) + set `--font-family-sans`
- Locale includes Japanese ‚Üí load Noto Sans JP font
- Locale includes Korean ‚Üí load Noto Sans KR font

**Set body classes** (if not already set):
```html
<body class="bg-background text-foreground">
```

**Verify setup is complete before writing any UI code:**
- [ ] `npx 7onic init` completed successfully
- [ ] Body has `bg-background text-foreground`
- [ ] Dark mode toggle (if selected)
- [ ] Font loaded (if custom)
- [ ] CJK fonts loaded (if applicable)

‚õî **Do NOT proceed to Step 3 until all checks pass.**

### Step 3: Add components as needed

```bash

npx 7onic add button card input

# Add all components at once
npx 7onic add --all

# Dependencies are resolved automatically (e.g., input ‚Üí input + field)
```

### Step 4: Add Toaster to root layout (if using Toast)

```bash
npx 7onic add toast
```

Place `<Toaster />` once in your root layout file. Without this, `toast()` calls will not render.

```tsx

// In your root layout (e.g., app/layout.tsx)

  {children}
  <Toaster position="bottom-right" />
</body>
```

### Step 5: Start building

Design freely based on user's project description. Always use 7onic components + tokens.

## ‚õî AI Rules ‚Äî Whitelist System

### Core Principle

**Token values are user-defined.** Every project has different brand colors and design decisions. The token NAMES are the API ‚Äî never assume or hardcode specific values.

### Whitelist ‚Äî ONLY These Are Allowed

1. **7onic components + Props** ‚Äî always prefer components over raw HTML
2. **Token classes** ‚Äî colors, spacing, typography, radius, shadows, z-index, icon sizes, duration, easing, opacity, scale
3. **Tailwind structural utilities** ‚Äî layout, positioning, display, overflow, sizing, and more:
   - Layout: flex, grid, block, inline, hidden, container
   - Position: relative, absolute, fixed, sticky, top-0, inset-0
   - Flex/Grid: items-center, justify-between, gap-4, col-span-2
   - Sizing: w-full, h-full, min-h-screen, max-w-7xl
   - Spacing: space-x-*, space-y-*, divide-x, divide-y
   - Overflow: overflow-hidden, overflow-auto, truncate, whitespace-nowrap
   - Interaction: cursor-pointer, pointer-events-none, select-none
   - Accessibility: sr-only
   - Group/Peer: group, group-hover:*, peer, peer-checked:*
   - Aspect: aspect-square, aspect-video
   - Text: line-clamp-*, text-ellipsis
   - Animation: animate-spin, animate-pulse, animate-bounce
   - Any other Tailwind structural/layout utility not listed above is also allowed ‚Äî as long as it does not hardcode colors, spacing values, font sizes, or other visual values that exist as tokens
   - ‚öÝÔ∏è Utilities that implicitly use color (divide, border) must be paired with a token color:
     `divide-y divide-border` ‚úÖ / `divide-y` alone ‚ùå (may not adapt to dark mode)
     `border border-border` ‚úÖ / `border` alone ‚ùå
4. **Tailwind visual utilities using token values** ‚Äî gradients, transforms, transitions:
   - Gradient: bg-gradient-to-r, from-primary, to-secondary (token colors only)
   - Transform: rotate-45, translate-x-1
   - Transition: transition-all, transition-colors (with token durations)
   - Backdrop: backdrop-blur, backdrop-blur-sm
5. **Responsive prefixes** ‚Äî sm:, md:, lg:, xl:, 2xl: (on allowed classes only)
6. **State prefixes** ‚Äî hover:, focus:, active:, disabled: (on allowed classes only, token values only)
7. **Layout dimension arbitrary values** ‚Äî height and width ONLY: h-[300px], max-w-[1200px] (when no token fits)
8. **Opacity modifier** ‚Äî append `/0-100` to any token color for transparency:
   - `bg-primary/50`, `text-foreground/70`, `border-border/30`
   - Do NOT use `opacity-*` utility as a workaround ‚Äî it affects the entire element including children
   - `bg-primary/10` ‚úÖ (only background is transparent) / `bg-primary opacity-10` ‚ùå (children also become transparent)

9. **Token-first rule** ‚Äî Before adding any Tailwind utility, check if the property is already included in a design token. Typography tokens (`text-sm`, `text-md`, etc.) include both font-size AND line-height as a pair. Do NOT override with Tailwind line-height utilities:
   - `text-sm` ‚úÖ (token provides font-size + line-height)
   - `text-sm leading-relaxed` ‚ùå (overrides token line-height)
   - `text-sm leading-none` ‚ùå (overrides token line-height)
   - `text-sm leading-[18px]` ‚ùå (hardcodes line-height)

**Everything not in this list is FORBIDDEN.**

‚öÝÔ∏è **Gradient/visual utilities must use token colors only:** `from-primary to-secondary` ‚úÖ / `from-blue-500 to-purple-600` ‚ùå

### Decision Tree ‚Äî For Every UI Element

```
Step 1: Does a 7onic component exist for this?
  ‚Üí YES: Use the component. Style via Props (variant/size/color).
         className ONLY for layout (margin, width, flex positioning).
  ‚Üí NO: Step 2

Step 2: Is this a layout/structural element? (flex container, grid, section wrapper)
  ‚Üí YES: Use div/section + token classes only.
  ‚Üí NO: Step 3

Step 3: Re-check the 42 components. Most UI can be built with component combinations.
  ‚Üí Still no match: Use div + token classes only.
```

### ‚ùå Forbidden Patterns

```tsx
// ‚ùå HTML instead of components
<button className="bg-primary rounded-lg px-4 py-2">   ‚Üí <Button>
<input className="border rounded-lg px-3 h-10">         ‚Üí <Input>
<div className="border rounded-xl p-6 shadow-sm">       ‚Üí <Card>
<table className="w-full">                               ‚Üí <Table>

// ‚ùå className overriding component styles
<Button className="bg-blue-500 text-white">   ‚Üí <Button color="primary">
<Badge className="bg-red-100 text-red-700">   ‚Üí <Badge color="error">

// ‚ùå Raw Tailwind colors
bg-blue-500, text-gray-700, bg-white, border-gray-200

// ‚ùå Dark mode prefix
dark:bg-gray-900, dark:text-white

// ‚ùå Arbitrary values (except layout dimensions)
p-[17px], text-[13px], rounded-[7px], z-[999]

// ‚ùå Inline styles
style={{ color: '#333', padding: '20px' }}

// ‚ùå @apply with raw values
@apply bg-blue-500

// ‚ùå Icon sizing with w/h
w-4 h-4, w-5 h-5   ‚Üí icon-sm, icon-md

// ‚ùå Radix direct import
import * as Dialog from '@radix-ui/react-dialog'   ‚Üí import { Modal } from '@/components/ui/modal'

// ‚ùå Unnecessary component wrappers
function MyButton(props) { return <Button {...props} /> }   ‚Üí Use Button directly
```

If the user needs custom values beyond existing tokens, they will handle it separately.

## Dark Mode ‚Äî Automatic

Dark mode works automatically when using semantic tokens. No `dark:` prefix needed.

| ‚ùå NEVER | ‚úÖ CORRECT |
|---|---|
| `bg-white dark:bg-gray-900` | `bg-background` |
| `text-gray-900 dark:text-gray-100` | `text-foreground` |
| `border-gray-200 dark:border-gray-700` | `border-border` |

### Dark Mode Toggle Implementation

Strategy: `dark` class on `<html>` element.

**Required behavior:**
1. Toggle adds/removes `dark` class on `<html>`
2. Persist choice to localStorage (key: `"theme"`)
3. On page load: check localStorage first ‚Üí fallback to system preference
4. System preference: `window.matchMedia('(prefers-color-scheme: dark)')`

**Three states:** `'light'` | `'dark'` | `'system'`

**Minimal logic:**
- Read: `localStorage.getItem('theme') || 'system'`
- Apply: `document.documentElement.classList.toggle('dark', isDark)`
- Save: `localStorage.setItem('theme', newTheme)`

**Add a toggle button in the header** that cycles through light ‚Üí dark ‚Üí system (or light ‚Üî dark).

| Metric display | MetricCard |
| Tag / chip | Badge (with `removable`) |

## Compound Component Structure

Compound components have **required sub-components**. Missing them causes errors or broken UI.

**Modal ‚Äî required structure:**
```tsx
<Modal>
  <Modal.Trigger>...</Modal.Trigger>
  <Modal.Portal>           {/* ‚Üê Required for correct z-index */}
    <Modal.Overlay />      {/* ‚Üê Required for backdrop */}
    <Modal.Content>
      <Modal.Header>
        <Modal.Title>...</Modal.Title>
      </Modal.Header>
      <Modal.Body>...</Modal.Body>
    </Modal.Content>
  </Modal.Portal>
</Modal>
```

**Drawer ‚Äî same pattern as Modal** (Portal + Overlay + Content required)

**Tooltip ‚Äî Provider required:**
```tsx
<Tooltip.Provider>         {/* ‚Üê Required, wrap app once */}
  <Tooltip>
    <Tooltip.Trigger>...</Tooltip.Trigger>
    <Tooltip.Portal>
      <Tooltip.Content>...</Tooltip.Content>
    </Tooltip.Portal>
  </Tooltip>
</Tooltip.Provider>
```

**Toast ‚Äî imperative API, no compound structure:**
```tsx
// Place Toaster once in layout
<Toaster />
// Call toast() anywhere
toast.success('Done')
```

## asChild Pattern (Slot)

`asChild` replaces the component's DOM element with its child, keeping styles and behavior:

```tsx
// ‚ùå Double <a> ‚Äî broken HTML
<Link href="/about">
  <Button>About</Button>
</Link>

// ‚úÖ Button renders as <a> with Button styles
<Button asChild>
  <Link href="/about">About</Link>
</Button>

// ‚úÖ DropdownMenu item as link
<DropdownMenu.Item asChild>
  <Link href="/settings">Settings</Link>
</DropdownMenu.Item>

// ‚úÖ Breadcrumb as Next.js Link
<Breadcrumb.Link asChild>
  <Link href="/">Home</Link>
</Breadcrumb.Link>
```

**Use `asChild` when combining 7onic components with routing `<Link>`.**
Components that support `asChild`: Button, IconButton, Badge, Card, Modal.Trigger, Modal.Close, Drawer.Trigger, Drawer.Close, DropdownMenu.Item, Breadcrumb.Link, NavigationMenu.Link, Tooltip.Trigger, Popover.Trigger

---

## Props Usage

**Always use allowed values.** Don't pass arbitrary strings to variant/size/color props.

```tsx
// ‚úÖ Correct
<Button variant="solid" size="default" color="primary" />

// ‚ùå Wrong ‚Äî invalid values
<Button variant="primary" />     // 'primary' is a color, not a variant
<Button size="medium" />         // Use 'md' or 'default', not 'medium'
<Button color="blue" />          // Use 'primary', not raw color names
```

**`className` on components = layout ONLY** (margin, width, flex positioning). All visual styling (color, size, radius, variant) must use Props.

**Style via Props, not className:**
```tsx
// ‚úÖ Use Props
<Button color="primary" size="lg" />
<Badge color="error" variant="subtle" />

// ‚ùå Don't override with className
<Button className="bg-blue-500 text-white" />
<Badge className="bg-red-100" />
```

## Form Patterns

**Use Field wrapper** for labeled form elements with validation:

```tsx
import { Field } from '@/components/ui/field'
import { Input } from '@/components/ui/input'
import { Textarea } from '@/components/ui/textarea'
import { Checkbox } from '@/components/ui/checkbox'

{/* Text input with label + error */}
<Field error={errors.email}>
  <Field.Label required>Email</Field.Label>
  <Input placeholder="you@example.com" />
  <Field.Error />
</Field>

{/* Textarea with character count */}
<Field>
  <Field.Label>Bio</Field.Label>
  <Textarea maxLength={200} />

</Field>
```

**Field context auto-connects** to Input, Textarea, Select, Checkbox, RadioGroup:
- Passes `error` state
- Passes `disabled` state
- Auto-generates matching `id` / `htmlFor` / `aria-describedby`

## Performance

**Components are local files ‚Äî no tree-shaking needed.** Only the components you add are in your project.

```bash
# Only adds button and its dependencies
npx 7onic add button
```

**Don't wrap components unnecessarily:**
```tsx
// ‚ùå Unnecessary
function MyButton(props) { return <Button {...props} /> }

// ‚úÖ Use directly
<Button variant="solid" color="primary">Submit</Button>
```

**Next.js App Router:** 7onic components have `'use client'` internally ‚Äî they render in Server Components.
But if YOUR code uses React hooks (`useState`, `useEffect`) or event handlers (`onClick`, `onChange`), add `'use client'` at the top of your file. When in doubt, add `'use client'` ‚Äî it's always safe.
